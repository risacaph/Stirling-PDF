Authentication & Login

By default Papyra can run with or without login. Enable login to get user accounts, roles, teams, and access plans.

Enable login

SECURITY_ENABLELOGIN=true

On first boot an admin account is created. Seed your own credentials before the first start:

SECURITY_INITIALLOGIN_USERNAME=admin
SECURITY_INITIALLOGIN_PASSWORD=change-me-please
If you don't set these, the default is admin / stirling with a forced password change on first login.

Lockout protection

  • security.loginAttemptCount — lock an account after this many failed attempts (default 5; set -1 to disable, e.g. when using Fail2Ban).
  • security.loginResetTimeMinutes — how long the lock lasts (default 120).
Locked out during testing? The lockout is in-memory — restarting the container clears it. An admin can also unlock a user from Settings → People.

Single sign-on

Papyra supports OAuth2 / OIDC (Keycloak, Google, GitHub, or any OpenID Connect provider) and SAML2. Set security.oauth2.enabled or security.saml2.enabled to true (login must also be enabled), then configure your provider's client ID, secret, and issuer.

For SSO to work you must set SYSTEM_BACKENDURL to your public URL — that's where your identity provider sends callbacks. For SAML, download the SP metadata from the backend URL (/saml2/service-provider-metadata/{registrationId}).

autoCreateUser creates accounts on first SSO login; blockRegistration requires an admin to pre-create the account first.
