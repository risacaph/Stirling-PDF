Configuration Overview

Papyra reads its configuration from a settings.yml file inside the /configs volume. Almost every setting can also be overridden with an environment variable — which is the usual approach for Docker deployments.

Environment variable naming

Take the dotted path of a setting, upper-case it, and join with underscores. For example:

SettingEnvironment variable
security.enableLoginSECURITY_ENABLELOGIN
security.initialLogin.usernameSECURITY_INITIALLOGIN_USERNAME
system.defaultLocaleSYSTEM_DEFAULTLOCALE
system.backendUrlSYSTEM_BACKENDURL
Don't comment out entries in settings.yml — commented lines are removed on the next startup. Leave a setting at its default instead.

Main sections

  • security — login, lockout, OAuth2 / SAML2, JWT. See Authentication & Login.
  • system — locale, analytics, CORS, backend/frontend URLs, mobile scanner. See System Settings.
  • mail — SMTP for invites and password resets.
  • storage / processExecutor / customPaths — temp storage, tool concurrency, and binary paths. See Advanced Configuration.
  • endpoints — enable/disable specific tool endpoints.
  • mcp / aiEngine / policies — MCP server, AI features, and document policies.
In this build, the vendor premium settings are effectively ignored — all premium/enterprise capabilities are on regardless. Per-user access is controlled through Access Plans instead.
